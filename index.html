Spreadsheet Polisher

I created....read, vibe coded.... this little tool. A buffer that takes spreadsheets formatted for people and turns it into clean, typed data for ingestion into BI tools and warehouses. I had the idea because I have had so many situations where a sheet comes in that is great for communication but very poortly structured for using in data engineering. I wanted a tool for my personal toolbelt for the reality of an Excel-heavy org that receives the same messy .xlsx every month and needs a faster way of converting it for use in their pipelines. Sure I could just throw the sheet at the ai every time, but if I build a tool like this, we can retain data privacy and I can maybe save some tokens over time.

Python pandas openpyxl pandera pyarrow Streamlit Claude

The core idea

The LLM never touches the numbers. Given a messy spreadsheet, Claude proposes a cleaning recipe — a reviewable YAML config. A human approves it. Then a deterministic engine executes the approved recipe. The durable artifact is the versioned recipe, so a recurring, same-shaped file replays with zero LLM calls and full reproducibility. The LLM only re-engages when the file drifts and validation fails.

Propose (LLM, once per new file shape) cruddy.xlsx → extract (cells + formatting) → profile → Claude proposes recipe.yaml → human reviews / edits / approves Execute (deterministic, every run) file + approved recipe.yaml → locate → normalize → map → validate → out/: clean_<file>.parquet + .csv + schema.json + quality_report.md + audit_log

What "cruddy Excel" means here

  • Structural — title/logo rows, multi-row/merged headers, interleaved subtotals, blank spacers, multiple tables per sheet, one-sheet-per-month, pivot-wide layouts.
  • Value/type — numbers-as-text ("1,234", "$1,200", "(500)", "12%"), text/serial dates, null tokens (N/A, -, #REF!), units in cells, mangled IDs.
  • Semantic — synonym columns (VIN / Vin # / Vehicle ID), categorical drift, duplicate rows.
  • Meaning-in-formatting — red fill = over budget, bold = subtotal, cell comments.

Two surfaces, one library

A CLI automates the execute path (profile → run → propose); a Streamlit app covers the review/approve path — pick a file, see the profile, propose or load a recipe, watch the dry-run live, approve into recipes/.

Principles

  • Nothing is silently guessed — low-confidence items surface in the quality report.
  • Every transform is logged, so an output number traces back to a source cell.
